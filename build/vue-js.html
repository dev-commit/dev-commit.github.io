Динамические компоненты

Создание компонента

<component :is="currentView"></component>

Кэширование компонента (сохранение состояния)

<keep-alive> <component :is="currentView"></component> </keep-alive>

Пример

<!-- если нужна анимация, component обернуть в transition --> <div id="app"> <nav> <a href="#" @click="changeView('orange')">orange</a> <a href="#" @click="changeView('blue')">blue</a> </nav> <component :is="currentView"></component> </div> <template id="orange-template"> <div class="orange-block"></div> </template> <template id="blue-template"> <div class="blue-block"></div> </template> const app = new Vue({ el: '#app', data: { currentView: 'orange' }, components: { orange: { template: '#orange-template' }, blue: { template: '#blue-template' } }, methods: { changeView(view) { this.currentView = view } } })

Computed Properties

  • Сеттеры вычисляемых свойств
  • Примеры
  • Computed Properties - вычисляемые (обработанные) свойства применяются когда есть зависимость переменных от других переменных
  • Кэшируются, основываясь на своих реактивных зависимостях. Вычисляемое свойство пересчитывается лишь тогда, когда изменится одна из его реактивных зависимостей
<div id="app"> <input type="text" @input="increment($event.target.value)"> <p>{{value}}</p> <p>{{doubleValue}}</p> </div> const app = new Vue({ el: '#app', data: { value: 1 }, methods: { increment(value) { this.value = value } }, computed: { doubleValue() { return this.value * 2 } } });

Сеттеры вычисляемых свойств

  • По умолчанию вычисляемые свойства работают только на чтение, но при необходимости можно также указать и сеттер
  • Теперь запись vm.fullName = 'Иван Иванов' вызовет сеттер, и vm.firstName и vm.lastName будут соответствующим образом обновлены
computed: { fullName: { // геттер: get: function () { return this.firstName + ' ' + this.lastName }, // сеттер: set: function (newValue) { let names = newValue.split(' ') this.firstName = names[0] this.lastName = names[names.length - 1] } } }

Примеры

Пример #1. При вводе более 10 символов, скрывается кнопка

<div id="app"> <form action=""> <span>{{ count }} / 10</span> <textarea v-model="text"></textarea> <button v-if="count <= 10">Отправить</button> </form> </div> const app = new Vue({ el: '#app', data: { text: '' }, computed: { count() { return this.text.length } } })

Передача событий между компонентами

  • Создание событий. Глобальный метод $emit()

Прослушиване событий

<app-child @close="modal = false"></app-child> <!-- родительский компонент --> <button @click="$emit('close')"></button> <!-- дочерний компонент --> <app-child @close="modal = $event"></app-child> <!-- родительский компонент --> <button @click="$emit('close', true)"></button> <!-- дочерний компонент -->

Пример

<div id="app"> <ul> <li v-for="message in messages">{{ message }}</li> </ul> <message @message-saved="handleMessage"></message> </div> Vue.component('message', { template: '<input type="text" v-model="message" @keyup.enter="saveMessage">', data: function() { return { message: '' } }, methods: { saveMessage() { this.$emit('message-saved', this.message); // передача события родителю this.message = ''; } } }) const app = new Vue({ el: '#app', data: { messages: [] } methods: { handleMessage(message){ this.messages.push(message) } } })

.fade-enter-active { animation: fade 0.4s; } .fade-leave-active { animation: fade 0.4s reverse; } @keyframes fade { 0% { transform: scale(0); } 50% { transform: scale(1.5); } 100% { transform: scale(1); } }
