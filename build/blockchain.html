Blockchain

Структура блока

Blockchain - цепочка блоков, которые содержат информацию:

  1. Данные. Зависит от типа Blockchain, н-р для Bitcoin: отправитель, получатель, сумма
  2. Хэш блока. Если в блоке есть изменения, то меняется хэш блока, что делает недействительным все последующие блоки (каждый последующий блок хранит хэш предыдущего)
  3. Хэш предыдущего блока. Если нет предыдущего блока, то это первый блок (Генезис-блок)

Децентрализация

  • Использование одноранговой сети
  • Децентрализованный глобальный код, открытый для всех

P2P (Peer-to-Peer). Blockchain использует P2P сеть и любой человек может присоединиться

  • При подключении к сети, пользователь получает полную копию Blockchain
  • При создании нового блока, он отправляется всем подключенным к сети узлам (пользователям). Каждый узел проверяет новый блок и его хэш, чтобы убедиться в его достоверности. Если всё в порядке, то каждый узел добавляет это блок в свою копию Blockchain. Если блок фальсифицирован, то будет отклонен остальными узлами сети
  • Чтобы фальсифицировать блок, то под контролем должно быть более 50% сейти Blockchain, чтобы признать консенсус и признать фальсифициорованный блок действительным, поэтоум однажды записанную информация практически невозможно изменить или удалить

Принципы безопасности

  1. Хэширование - задание уникального хэша каждому блоку. Блок хранит свой хэш и хэш последующего блока
  2. Prof of Work - механизм, замедляющий создание новых блоков (для Bitcoin это 10 минут). Существенно затрудняется изменение блоков: если изменяется 1 блок, то нужно пересчитать Prof of Work для всех остальных блоков, что является труным процессом
  3. Децентрализация

Создание нового блока

  • Эмиссия осуществляется за счёт майнинга
  • Устройство, которые быстрее всех решит криптографическую задачу и создаст новый блок, получит немного криптовалюты

Правила

  • У каждой криптовалюты должен быть свой Blockchain

Smart Contract (Смарт-контракт)

  • Smart Contract - программа написанная на языке Solidity, хранящиеся внутри Blockchain и способная выполнять определенные действия (автоматический обмен монет и т.д.)
  • Программа проверяет корректность выполнения всех условий между продавцом и покупателем. Если корректно, то сделка совершается - иначе, откатывается

Solidity

  • Solidity - объектно-ориентированный язык высокого уровня для реализации Smart Contracts, которые можно воспринимать как API
  • Smart Contracts можно воспринимать как API
  • OpenZeppelin - набор готовых Smart Contracts

Crypto Wallet

  • Crypto Wallet (криптокошелёк) - физически НЕ хранят криптовалюту и НЕ отправляют крипто-токены, вместо этого у кошелька есть ключи к криптовалюте, которая хранятся в общедоступных сетях Blockchain

Задачи

  1. Предоставляют интерфейс для управления криптовалютными балансами
  2. Предоставляют цифровые подписи, авторизующие каждую транзакцию
  3. Хранят Public Key и Private Key, необходимые для покупки криптовалюты
  4. Некоторые кошельки позволяют выполнять операции с криптовалютой: покупать, продавать, взаимодействовать с децентрализованными приложениями Decentralised Applications (dApps)

Открытый и закрытый ключ

  • Public Key (открытый ключ) - подобен номеру банковского счета и может широко распространяться
  • Позволяет пользователям получать транзакции криптокошелька
  • Private Key (закрытый ключ) - подобен паролю банковского счета или PIN-коду и должен храниться в секрете
  • Private Key пользователя подтверждает право собственности на соответствующий Public Key
  • Каждый Public Key связан с одним соответствующим Private Key: вместе они используются для шифрования и расшифровки данных
  • При отправке токенов Private Key пользователя подписывает транзакцию и передает ее в сеть Blockchain

Типы кошельков

Hot Wallet

  • Hot Wallet (горячие кошельки) - подключены к Интернету
  • Private Key хранятся и шифруются в самом приложении, которое хранится в сети
Примеры
  • Web-based wallets
  • Mobile wallets
  • Desktop wallets

Cold Wallet

  • Cold Wallet (холодные кошельки) - находятся в автономном режиме
Примеры
  • Paper wallet - физическое место, где ключи пользователя записываются или распечатываются
  • Hardware wallet - внешний девайс (обычно устройство USB или Bluetooth), в котором хранятся ключи пользователя. Пользователь может подписать транзакцию, только нажав физическую кнопку на устройстве

Custodial

  • Custodial - большинство сетевых криптокошельков, как правило, являются кастодиальными кошельками. Обычно предлагаемые на биржах криптовалюты
  • Пользователи больше не имеют полного контроля над своими токенами, а Private Key, необходимые для подписи транзакций, хранятся только на бирже

Non-Custodial

  • Non-Custodial - Private Key хранится локально у пользователя
  • Пользователя просят записать и безопасно сохранить список из 12 случайно сгенерированных слов, известных как recovery (восстановление), seed (начальное число) или mnemonic phrase (мнемоническая фраза)
  • Аппаратные кошельки по своей сути не связаны с хранением, поскольку закрытые ключи хранятся на самом устройстве

NFT Wallet

  • NFT Wallet - кошельки для ханения NFT
  • Для кошельков NFT существует два основных варианта: Hardware wallets (аппаратные) или Software-based wallets (программные)

Metamask

  • Metamask - отдельная программа / плагин для браузеров / крипто-кошелек
  • Позволяет авторизовать Web 3.0 приложение в Blockchain сети при помощи аккаунта
  • В MetaMask можно создать кошелек (аккаунт) Ethereum и использовать этот аккаунт для авторизации
  • Позволяет подключить Сhrome-браузер к сети Blockchain

web3auth Wallet

  • Web3 Wallet - инфраструктура для подключения цифровых кошельков

NFT

  • NFT (Non-Fungible Token, невзаимозаменяемый токен) - цифровой сертификат, подтверждающий факт владения конкретным цифровым активом
  • В блокчейн-сети можно найти информацию обо всех владельцах вплоть до текущего
  • Права NFT работают только в рамках блокчейн-сети, в которой был создан токен
  • Минтинг - создание NFT токена

Application Web 3.0

Два варианта работы с Blockchain

  • 1. Frontend Only. Frontend напрямую обращается к Blockchain со Smart Contracts как к API и получает данные
  • 2. Backend Only. Frontend работает с API сервера, не взаимодействуя с Blockchain напрямую. Application Server (Backend) взаимодействует с Blockchain и БД

Библиотеки Web3.js и Ethers.js

  • Web3.js - библиотека JavaScript для взаимодействия с Blockchain-сетью Ethereum или любым другим, построенным на базе Ethereum
  • Она превращает приложение (н-р на React) в приложение с поддержкой Blockchain
  • Ethers.js - библиотека для всех ваших потребностей Ethereum

Truffle

  • Truffle - фреймворк, предлагающий набор инструментов:
  1. Деплой приложния / миграция в сеть Ethereum
  2. Сборка и управление Smart Contracts
  3. Создание аккаунтов в Ethereum
  4. Управление сетью
  5. Консоль разработки
