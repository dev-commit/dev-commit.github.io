Структура Node.js

  • Node.js является Middleware между нашим кодом и ОС

Общая структура

  • libuv - приносит абстракцию над системный Демультиплексором. Т.к. в каждой система свой Демультиплексор (Linux, Windows и т.д.)

Event Loop

  • Event Loop - позволяет Node.js выполнять асинхронные неблокирующие I/O (Input / Output) операции, несмотря на то что, JS является однопоточным. Путём выгрузки части операций в ядро, если это возможно.
  • I/O - input/output операции
  • setImmediate - применяется чтобы поставить функцию в очередь на выполнение без задержки. У setImmediate единственный аргумент – это функция, выполнение которой нужно запланировать (callback)
  • Object.observe() - позволяет асинхронно отслеживать изменения состояния JavaScript объектов во времени, без использования каких-либо сторонних библиотек

Event Loop

libuv - приносит абстракцию над системный Демультиплексором. Т.к. в каждой система свой Демультиплексор (Linux, Windows и т.д.)
    Event Loop
  • Event Loop обращается в "Очередь событий", берёт событие, идёт в "Приложение" и вызывает callback "Обработчик"
  • "Приложение" может выполнить "Обработчик" или запустить новый асинхронный запрос
  • При отсутствии собыйтий в "Очереди событий", Event Loop возвращает управление в "Демультиплексор событий"
  • Программа заканчивается, когда ничего нет в "Очередь событий" и "Демультиплексор событий"

Event Loop structure

Фазы

    1. Timers
  • Попадают callback setTimeout и setInterval
  • Для Libuv setTimeout и setInterval - это одна функция, но у setInterval есть параметр repeat
    • 2. Pending Callbacks
  • Очередь куда попадают I/O операции (не все): connection к сети, коллбэки ошибок
    • 3. Idle, Prepare
  • Нет доступа к этой фазе. Только Event Loop может попасть туда, н-р: перед тем как начинает читать файл, Event Loop может подготовить какие-то данные, то попадает в эту фазу
    • 4. Poll
  • Попадают I/O операции: начало чтение файла
    • 5. Check
  • Попадают только setImmediate
  • setImmediate применяется чтобы поставить функцию в очередь на выполнение без задержки. У setImmediate единственный аргумент – это функция, выполнение которой нужно запланировать (callback). Это callback попадает в Check
    • 6. Close Callbacks
  • Попадают callback вида "close": onClose, socketOnClose и т.д.

Thread Pool

  • Thread Pool - используется для запуска пользовательского кода и получения уведомление в потоке цикла (происходит возврат обраноно в Loop)

Установка кол-ва потоков

  • Можно контролировать количество потоков, установив process.env.UV_THREADPOOL_SIZE
  • process.env.UV_THREADPOOL_SIZE == 4 (по умолчанию). Максимальное значение 128
  • Нет смысла устанавливать кол-во потоков большее, чем кол-во ядер в процессоре

libuv

  • libuv - многоплатформенная библиотека C, которая обеспечивает поддержку асинхронного ввода-вывода на основе Event Loops. Он поддерживает порты событий epoll, kqueue, Windows IOCP и Solaris. Он в первую очередь предназначен для использования в Node.js, но также используется в других программных проектах

Node.js Stream

  • Stream - абстрактный интерфейс для работы с потоковыми данными в Node.js. Модуль node:stream предоставляет API для реализации потокового интерфейса
  • Стрим — это концепция, c помощью которой можно обрабатывать данные небольшими частями, что позволяет задействовать небольшой объем оперативной памяти. Также с ее помощью можно разбить обработку каждой части на независимые друг от друга модули (функции либо классы). Например, можно сразу сжать часть данных, потом зашифровать и записать в файл. Основная идея в том, чтобы не работать с данными целиком, а поочередно обрабатывать часть данных

Информация

Stream VS Thread

  • В контексте Node.js, Streams позволяет работать с потоковыми данными, а Thread относится к многопоточности (когда что-то может выполняться параллельно), например - Worker Threads

4 вида Stream в Node.js

  • Readable - чтение
  • Writable - запись
  • Duplex - чтение и запись
  • Transform - вид Duplex потока, который может изменять данные
const { Readable } = require('stream'); // Readable stream const { Writable } = require('stream'); // Writable stream const { Duplex } = require('stream'); // Duplex stream const { Transform } = require('stream'); // Transform stream

Модульная система

Стандарты

1. Паттерн "Модуль" 2003 год

  • Немедленно вызываемая функция, образующая приватную область видимости

2. CommonJS (CJS) 2009 год

  • module.exports -> require
  • Реализация: Модульная система Node.js
  • Предназначен для синхронной загрузки. Преимущественно используется на стороне сервера

3. Asynchronous Module Definition (AMD) 2009 год

  • define -> require
  • Реализация: RequireJS
  • Предназначен для асинхронной загрузки. Преимущественно используется на стороне клиента

4. Universal Module Definition (UMD) 2011 год

  • Объединение AMD и CommonJS
  • Суть подхода: проверить существование функции define и если она есть определить модуль в формате AMD, а если нет, в формате CommonJS

5. Модули ES6 2015 год

  • export -> import

Модули ES6

    Особенности
  • Не блокируют парсинг html
  • Выполняются отложенно
  • Имею свою область видимости
  • Код модуля всегда выполняется в строгом режиме
  • Для экспорта обязательно нужно имя

  • Node.js будет рассматривать файлы .cjs как модули CommonJS, а файлы .mjs — как модули ECMAScript
