Информация

Инструменты

  • NPM - Node.js Package Manager. менеджер пакетов, входящий в состав Node.js
  • NPX - инструмент запуска NPM пакетов, доступный в версиях npm 5.2 и выше

Команды

Команды

Инициализация

npm init # инициализация проекта (создание package.json) npm init -y # инициализация проекта с параметрами по умолчанию

Установка пакетов

npm i # установить все пакеты из package.json npm i PackageName # установить package (не должен добавляться в package.json) npm i --save PackageName # установить package и добавить запись в секцию dependencies package.json npm i --save-dev PackageName # установить package и добавить запись в секцию devDependencies package.json npm i --g PackageName # установить package глобально

Обновление пакетов

npm up # обновить все пакеты из package.json npm up PackageName # обновить конктерный пакет package npm i -g npm-check-updates # проверка обновлений всех пакетов в package.json

Удаление пакетов

npm r PackageName # удалить package

Проверка версий зависимостей

npm view PackageName # вывод списка версий пакета npm view PackageName versions # вывод текущей версии пакета npm outdated # вывод текущей и новой версии всех пакетов

.npmrc

npm set registry http://myrepositiryurl # установить registry в .npmrc npm get registry # получить registry из .npmrc npm config ls -l # содержимое файла .npmrc

Очистка кэша

  • clean - удаляет все данные из папки кеша
npm cache clean --force # очистить кэш в npm npm cache verify # проверить кэш

Прочее

npm i npm # обновление npm npm -v # проверить версию npm npm run PackageName # запустить package ncu # проверить какие пакеты устарели ncu -u # обновить версию в package.json

Сокращения

install | i uninstall | r update | up --save | S --save-dev | D config | c list | ls

Установка версии пакета

npm i PackageName # npm i PackageName@4.16.1 # конкретная версия npm i PackageName@^ # последняя версия
  • ~ (тильда) подбирает последнюю минорную версию пакета (последнюю цифру), например ~7.3.3 найдет последнюю 7.3.x, допустим 7.3.12 но не 7.4.0
  • ^ (крышка) подбирает последнюю мажорную версию пакета (среднюю цифру), например ^7.3.3 найдет последнюю 7.x.x, допустим 7.4.0 но не 8.0.0
npm i PackageName
  • Пакета нет в package.json - установится последняя версия
  • Пакет есть в package.json PackageName": "1.0.0" - установится версия 1.0.0
  • Пакет есть в package.json PackageName": "^1.0.0" - установится последняя версия

json-server

  • API мок-сервер
  • Открывается на указанном порту, после этого можно отправдять запросы
npm i -g json-server

Файлы

package.json { "scripts": { "mock": "json-server -w db.json -p 4200 -d 450" }, } › w (watch) # Наблюдение за файлом "db.json" › p (port) # Открытие JSON-сервера на заданном 4200 порту › d (delay) # Зедежка. Для имитации работы реальных запросов db.json { "posts": [ { "id": 1, "title": "Title1" }, { "id": 2, "title": "Title2" }, ], "users": [ { "id": 1, "title": "User1" }, { "id": 2, "title": "User2" }, ] }

Запуск скрипта

1. Запуск скрипта npm run mock 2. Доступные URL (Resources) http://localhost:4200/posts http://localhost:4200/users 3. Отправка запроса из приложения fetch('http://localhost:4200/users');

npm-check-updates

  • Установка глобально
npm i -g npm-check-updates

Команды

  • ncu - Отображает в терминале новые версии пакетов
  • ncu -u - Отображает в терминале новые версии пакетов и обновляет файл package.json
  • После выполнения "ncu" нужно переустановить зависимости через npm i

concurrently

  • Позволяет запускать параллельно несолько скриптов
  • Это может быть несколько локальных серверов, запущенных параллельно
npm i --save-dev concurrently

Файлы

package.json
  • В примере параллельно запускается толькольный сервер для Next.js и для json-server
{ "scripts": { "dev": "next dev", "mock": "json-server -w db.json -p 4200 -d 450", "serve": "concurrently \"npm run dev\" \"npm run mock\"" }, "dependencies": { "next": "12.0.7", }, "devDependencies": { "concurrently": "^6.4.0", } }

Скрипты

{ "scripts": { "hello": "echo \"hi\"", # вывод в консоль "start": "node index.js", # вызов js "server": "node ./node_modules/http-server/bin/http-server", # вызов npm-модуля http-server "server": "http-server", # аналогично "webpack": "webpack -w" # watcher } } node # выполнение кода в консоли node index.js # выполнение файла index.js npm start # выполнение скрипта из package.json npm server # выполнение скрипта из package.json (аналогично) npm run server # выполнение скрипта из package.json (аналогично)

Виды зависимостей

  • devDependencies - пакеты, которые нужны для разработки
  • dependencies - пакеты, от которых проект зависит непосредственно (библиотеки)
  • devDependencies и dependencies - устанавливаются в node_modules
  • При вызове npm install, npm установит пакеты указанные в dependencies, но не в devDependencies
  • Если сделать npm install внутри папки, в которой есть package.json, то установятся и те, и другие
