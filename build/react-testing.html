Информация

  • React Testing Library - ешение для тестирования компонентов React
  • Библиотека построена поверх библиотеки тестирования DOM (Testing Library DOM), в которой находится большая часть логики запросов
  • Вместо того, чтобы иметь дело с экземплярами визуализированных компонентов React, ваши тесты будут работать с реальными DOM-узлами
  • Эта библиотека является заменой для Enzyme
  • Библиотека не является Test Runner или Framework
  • Для работы рекомендуется Jest, хотя библиотека работает с любой средой

Установка

npm i --save-dev @testing-library/react npm i --save-dev @testing-library/jest-dom npm i --save-dev @testing-library/user-event
  • react-testing-library has moved to @testing-library/react
  • jest-dom has moved to @testing-library/jest-do

Методы

import { render, fireEvent, waitFor, screen } from '@testing-library/react';
  • render - метод рендерит элемент React в DOM и возвращает служебные функции для тестирования компонента.
  • fireEvent - метод позволяет запускать события для имитации действий пользователя
  • waitFor
  • screen
const { getByText, container, asFragment, } = render(<App />);

Примеры

  • После запуска теста, создается файл ./__snapshots__/App.test.js.snap в котором находится HTML-код компонента App
  • Если изменить HTML-код компонента App, то тест не пройдёт

import '@testing-library/jest-dom/extend-expect';

Test

import React from 'react'; import { render, cleanup } from '@testing-library/react'; import App from './App'; // для рендера одного компонента несколько раз afterEach(cleanup); it('Get Snapshot', () => { const { asFragment } = render(<App label="My Label" />); expect(asFragment()).toMatchSnapshot(); }); it('Check Element', () => { const { getByTestId, getByText } = render(<App label="My Label" />); // элемент содержит проп data-testid="headerId" const a1 = expect(getByTestId('headerId')); // элемент содержит проп data-testid="headerId", получаем атрибут "title" const a2 = expect(getByTestId('headerId').getAttribute('title')); // элемент содержит текстовый контент (полный, регистрозависимый) const a3 = expect(getByText('My Label')); // элемент содержит текстовый контент (допускается часть) const a4 = expect(getByText(/label/i)); // элемент содержит часть текстового контента (регистрозависимый) a1.toHaveTextContent('My'); // элемент содержит часть текстового контента (регистроНЕзависимый) a1.toHaveTextContent(/my/i); // элемент создержит className="headerClass" a1.toHaveClass('headerClass'); // элемент toBeInTheDocument a1.toBeInTheDocument(); // атрибут равен "Hello" a2.toBe('Hello'); });

Component

import React from 'react'; const App = ({ label }) => ( <header> <h1 data-testid="headerId" className="headerClass" title="Hello" > {label} </h1> </header> ); export default App;

Информация

  • react-test-renderer - пакет с рендерером, ктороый умеет рендерить React-компоненты в обычные JavaScript-объекты, не используя при этом DOM или мобильное окружение
  • Пакет react-test-renderer облегчает получение снимка иерархии представления платформы (чем-то похожего на DOM-дерево), отрендеренного компонентом React DOM или React Native. При этом не используются ни браузер, ни jsdom

Установка

npm i --save-dev react-test-renderer npm i --save-dev jest

Примеры

Создание Snapshot

import React from 'react'; import renderer from 'react-test-renderer'; const App = () => (<Button label="click me" />); const Button = ({ label }) => ( <div data-testid="button" className="button-style"> {label} </div> ); it('matches snapshot', () => { const tree = renderer.create(<Button lable="save"></Button>).toJSON(); expect(tree).toMatchSnapshot(); });
