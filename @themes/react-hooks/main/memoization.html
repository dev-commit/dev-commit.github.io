useCallback vs useMemo

Проблематика

  • В обычном состоянии при каждом ререндере компонента создается новая ссылка на функцию
  • Если функция используется в качестве dependencies в useEffect, то будет перерендер
  • Если функция передается в качестве props в дочерний компонент, то будет перерендер дочернего компонента

Решение через useCallback

  • useCallback - мемоизирует ссылку на функцию, чтобы она НЕ пересоздавалась каждый раз при перерендере
  • А если ссылку мемоизировать и прокинуть в дочерний компонент, обернутый в React.memo, то у дочернего компонента не будет перерендера
  • Если не использовать useCallback, то даже с React.memo будет происходить перерендер дочернего компонента, т.к. каждый раз будет приходить новая ссылка
  • useCallback полезен, когда переписовка компонента занимает большое количество времени
