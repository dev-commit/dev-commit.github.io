Работа с метками
  • Git использует два основных типа тегов: легковесные и аннотированные
  • Легковесный тег - это что-то очень похожее на ветку, которая не изменяется - просто указатель на определённый коммит
  • Аннотированные - теги хранятся в базе данных Git как полноценные объекты. Они имеют контрольную сумму, содержат имя автора, его e-mail и дату создания, имеют комментарий и могут быть подписаны и проверены с помощью GNU Privacy Guard (GPG)
git tag v1 # создание легковесеого тега git tag -a R2021.4 -m 'Release 2021.4' # создание аннотированного тега

Алгоритм работы

1. Информация о метках

git tag # просмотр списка меток git show v1.0.0 # показать коммит, отмеченный меткой

2. Создание метки

npm version 1.0.0 # автоматически повышает версию и создаёт тег (сделать для первой версии релиза) git tag v1.0.0 -a -m 'description' # создание аннотированной ссылки git tag v1.0.0 # создание легковесной ссылки git tag v1.0.0 HashCommit # создание легковесной ссылки по номеру коммита

3. Push метки

git push origin v1.0.0 # запушить тег git push origin --tags # запушить все теги

4. Удаление метки

git tag -d v1.0.0 # удалить тег из локального репозитория git push origin :refs/tags/v1.0.0 # обновление удаленного тэга пустым значением, что при водит к его удалению git push origin --delete v1.0.0 # удалить тег из удалённого репозитория
