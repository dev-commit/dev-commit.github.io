Слияние веток
git merge develop # слияние "develop" в текущую ветку git merge feature develop # замёржить в "feature" из "develop" git merge --abort # отмена merge
  • Разрешение конфликтов разницы "develop" и "feature"
git checkout develop # перейти в develop, если в ней есть обновления git pull # обновить ветку develop git checkout feature # перейти в feature git merge develop # merge из develop в свою ветку. ветка feature станет актуальной
  • Если появятся конфликты в файлах - поправить их

Описание

  • При merge происходит создание нового merge-комита в "feature" ветке
  • merge-комит будет включать изменения в "develop" с момента откалывания "feature"
  • Merge не разрушает историю: поверх всех коммитов, которые были в ветке, добавляется еще 1 коммит с изменениями из "develop". Если над "feature" веткой работают другие разработчики, то проблем не возникнет
