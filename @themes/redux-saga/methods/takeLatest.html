takeLatest

  • takeLatest
  • Используется если внутренний/внешний экземпляр или пользователь интерфейса могут инициировать несколько последовательных действий и желателен только вывод последнего значения
  • Получает только ответ на последний запрос, который был запущен, и ее можно рассматривать как оболочку для `yield take`
  • Хорошим примером могут быть быстрые вызовы API-интерфейса брокера для получения текущего тикера стоимости акций, где интерес представляют только самые последние/самые последние значения
  • takeEvery не обрабатывает неупорядоченные ответы задач. Нет никакой гарантии, что задачи завершатся в том же порядке, в котором они были запущены. Чтобы обрабатывать ответы, не соответствующие порядку, вы можете использовать takeLatest
  • takeLatest вместо этого запускает новую задачу fetchUser для каждого отправленного действия USER_REQUESTED. Поскольку takeLatest отменяет любую ожидающую задачу, начатую ранее, мы гарантируем, что если пользователь быстро инициирует несколько последовательных действий USER_REQUESTED, мы завершим только последнее действие
import { takeLatest } from 'redux-saga/effects' yield takeLatest(showReloadAppModalAttempt.type, reloadAppModalAttempt)
