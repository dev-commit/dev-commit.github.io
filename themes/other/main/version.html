Политика версионирования

Принципы семантического версионирования (semver)

МАЖОРНАЯ.МИНОРНАЯ.ПАТЧ

  1. МАЖОРНАЯ версия - сделаны обратно несовместимые изменения API
  2. МИНОРНАЯ версия - добавлена новая функциональность, е нарушая обратной совместимост
  3. ПАТЧ-версия - сделаны обратно совместимые исправления
  4. Дополнительные обозначения для предрелизных и билд-метаданных возможны как дополнения к МАЖОРНАЯ.МИНОРНАЯ.ПАТЧ формату

Пример

    react: 16.3.0
  • 16 - мажорная версия (правки требующие рефакторинга когда, без обратной совместимости)
  • 3 - минорная версия (правки с обратной совместимостью)
  • 0 - патч версия (фиксы)
