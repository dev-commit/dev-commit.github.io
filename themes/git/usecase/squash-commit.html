Squash Commit

  • Хорошая практика сквошить коммиты в один, чтобы на задачку был один коммит
  • Вообще засквошить можно когда угодно, потом форс-пушем долить, но после того как проверен PR этого уже делать не нужно будет, все правки после ревью доливаются отдельными коммитами

Локальный репозиторий

git checkout develop # перейти в develop, если в ней есть обновления git pull # обновить ветку develop git checkout ветка # перейти в рабочую ветку git merge-base dev HEAD # номер коммита к которому нужно откатиться git reset номер_коммита # откатывает всё что он сделал до того момента как откололся с develop git add . # индексация изменений git commit -m 'Номер_таски: название таски' # commit git push origin HEAD # push

Удаленный репозиторий

git push --force origin HEAD
