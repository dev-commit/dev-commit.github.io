Можно взять все изменения, которые попали в коммиты на одной из веток, и повторить их на другой (применить поверх другой)

Алгоритм работы

  1. Находится общий предок для двух веток (на которой вы находитесь сейчас и на которую вы выполняете перемещение)
  2. Для каждого из коммитов в текущей ветке берётся его дельта и сохраняется во временный файл
  3. Текущая ветка устанавливается на тот же коммит, что и ветка, на которую выполняется перемещение
  4. Одно за другим применяются все изменения
  • Rebasing - перемещение
  • "Автоматизированный" cherry-pick. Он выполняет ту же работу, но для цепочки коммитов, тем самым как бы перенося ветку на новое место
git rebase master git rebase -i master
