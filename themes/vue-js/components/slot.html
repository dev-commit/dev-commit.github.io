Передача данных между компонентами (slot)

Описание

  • Слот имеет доступ к тем же свойствам экземпляра app.vue (т.е. к той же «области видимости»), что и остальная часть шаблона
  • Слот не имеет доступа к области видимости <app-task>
  • Всё в родительском шаблоне компилируется в области видимости родительского компонента; всё в дочернем шаблоне компилируется в области видимости дочернего компонента

v-slot:user  // полная запись
#user        // сокращенная запись

Использование

Родительский компонент


<app-task>
	<template v-slot:user>User Admin</template>
</app-task>

Дочерний компонент


<template>
    <div>
        <slot name="user">User Default</slot>
    </div>
</template>

Слоты с ограниченной областью видимости

  • Атрибуты, привязанные к элементу <slot>, называются входными параметрами слота

Родительский компонент


{{userGeneral}}
<app-task>
    <template v-slot:user="slotProps">
    	<!-- slotProps виден только внутри template -->
    	{{userGeneral = slotProps.user}}
    </template>
</app-task>

<script>
export default {
    data() {
        return {
            userGeneral: null
        };
    }
</script>

Дочерний компонент


<template>
    <div>
        <slot name="user" :user="userChild"></slot>
    </div>
</template>

<script>
export default {
    data() {
        return {
            userChild: 'Tony'
        }
    }
}
</script>
